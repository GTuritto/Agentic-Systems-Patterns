1. Decision Rights

[ ] Agent decisions, recommendations, and forbidden decisions are explicit.
[ ] Human or professional review requirements are named.
[ ] Irreversible, regulated, or liability-creating actions require approval.
[ ] Escalation path is defined for low confidence or conflicting evidence.

2. Evidence Policy

[ ] Authoritative sources are named.
[ ] Forbidden or insufficient sources are named.
[ ] Freshness rules are defined.
[ ] Citation or explanation format is defined.
[ ] Missing, stale, or conflicting evidence has a safe outcome.

3. Data and Privacy

[ ] Sensitive data classes are named.
[ ] Tenant, jurisdiction, role, and user boundaries are enforced.
[ ] Retention, redaction, and deletion rules are defined.
[ ] Domain-specific records are auditable.

4. Tools and Actions

[ ] Domain tools have typed schemas and risk classes.
[ ] Tool permissions match role and task.
[ ] Side effects are idempotent, approved, or compensatable.
[ ] Domain policy runs outside the prompt.

[ ] Reviewer override is captured as data.
[ ] Review decisions become eval or policy updates when appropriate.
